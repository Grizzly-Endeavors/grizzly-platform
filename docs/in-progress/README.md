# In-Progress Tracker

Active, multi-phase work that is underway but not yet finished. This is the one place transient "resume point" state lives, kept out of the root `CLAUDE.md` and the durable docs so those never carry dated status that rots.

**How this fits the rest of the docs:**

- **This tracker** answers *what is in flight right now and where did we leave off* — ephemeral, deleted when the thread completes.
- **[ADRs](../decisions/)** answer *why* a choice was made — permanent.
- **[Runbooks](../runbooks/)** answer *how to operate or recover* a live system — permanent.
- **GitHub issues** hold discrete bugs, blockers, and small follow-ups — a thread here links to them rather than duplicating them.

See [`INDEX.md`](INDEX.md) for the current list of threads. The working rules for adding, formatting, and (importantly) *closing out* threads live in [`CLAUDE.md`](CLAUDE.md) — the short version is: when the work is done, delete the thread and make sure the permanent record (ADR + runbook + merged code) is in place.
