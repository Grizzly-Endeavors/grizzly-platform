# Runbooks

Operator procedures for the live platform — how to deploy, drive, rotate, and recover the things that are already running. A runbook is the durable "how to operate this" companion to an [ADR](../decisions/)'s "why we built it this way." Reach for one when you need to *do* something to a running system, not decide something.

See [`INDEX.md`](INDEX.md) for the full list, grouped by domain. If you stand up something new that needs multi-step recovery or non-obvious operation, add a runbook here and a line to the index.
